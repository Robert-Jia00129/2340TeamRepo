Project Description

SwiftHire is a web-based recruiting platform designed to streamline the hiring process for both job seekers and recruiters. Built as a Georgia Tech student project, our platform addresses the common pain points in modern recruitment by providing intuitive tools that make finding, evaluating, and connecting with talent more efficient.

The platform serves two primary user groups: job seekers looking for their next opportunity, and recruiters seeking qualified candidates. We provide role-specific features including advanced search capabilities, location-based job discovery, application pipeline management, and integrated communication tools—all within a clean, responsive interface.

Our development follows Agile Scrum methodology across three sprints, prioritizing user-centered design principles and modern web technologies. The result is a pragmatic solution that reduces friction in the hiring process while maintaining the flexibility both candidates and recruiters need to make informed decisions.

System's Description

SwiftHire is a comprehensive recruiting platform designed with both job seekers and recruiters in mind. Our web application features an intuitive, responsive design that adapts to different screen sizes and user workflows. Below are the key screens and features that respond to our users' needs:

👤

Job Seeker Features

📍 Location-Based Job Discovery

Interactive map view allows job seekers to filter opportunities by distance from their current location. The map displays job postings as markers, with real-time distance calculations and filtering controls. Users can set radius preferences (5, 10, 25, 50 miles) and view commute estimates.

🔒 Privacy Control Dashboard

Comprehensive privacy settings panel where job seekers control their profile visibility. Users can toggle what information recruiters see (contact details, work history, skills, projects), set profile to private/public, and manage who can message them. Real-time preview shows how their profile appears to recruiters.

🎯 AI-Powered Job Recommendations

Smart recommendation engine analyzes user skills, experience, and preferences to suggest relevant opportunities. The dashboard displays personalized job matches with compatibility scores, highlights matching skills, and explains why each role was recommended. Users receive notifications for new matches.

💼

Recruiter Features

🔍 Advanced Candidate Search

Powerful search interface with multi-criteria filtering. Recruiters can search by skills (with AND/OR operators), location (city, state, radius), years of experience, education, and project keywords. Results display in a sortable table with relevance scores and quick-view candidate cards. Saved search functionality stores frequent queries.

📋 Kanban Pipeline Management

Visual pipeline board for organizing applicants through hiring stages (Applied → Screening → Interview → Offer → Hired/Rejected). Drag-and-drop interface moves candidates between stages. Each card shows candidate photo, key skills, and quick actions. Stage-specific automations send notifications when candidates advance.

🤖 Candidate Recommendations

AI-driven candidate matching suggests qualified applicants for job postings. System analyzes job requirements and automatically recommends candidates whose profiles align with needed skills, experience level, and location preferences. Recruiters see match percentages and can invite candidates directly.

💬

Communication Tools

📧 In-Platform Email Integration

Recruiters can send emails to candidates through the platform without leaving the interface. Email templates, scheduling, and tracking are integrated. System masks personal emails initially, giving candidates control over when to share direct contact information.

💬 Direct Messaging System

Real-time chat interface allows recruiters and candidates to communicate inside the platform. Features include read receipts, typing indicators, file attachments, and message history. Respects user privacy settings and allows blocking/reporting for safety.

Process Description

Our team followed an Agile Scrum methodology throughout development, emphasizing iterative delivery and continuous feedback. Below are the key principles and patterns that guided our solution:

🔄 Sprint Planning & Execution

We organized work into 3 sprints with clear goals. Each sprint began with planning sessions where we broke down user stories into actionable tasks. Daily standups kept the team aligned, and sprint reviews/retrospectives ensured continuous improvement.

  • Sprint 1: Basic features including posting and applying for jobs
  • Sprint 2: Candidate search and application pipeline management
  • Sprint 3: Miscellaneous features like CSV export and map-based filtering for jobs and candidates

🎨 Design Patterns & Principles

Component-Based Architecture: Built reusable UI components (cards, modals, forms) for consistency across features. Each component follows single-responsibility principle.

RESTful API Design: Clean separation between frontend and backend. Endpoints follow REST conventions (GET /jobs, POST /messages, etc.) with consistent JSON responses.

Role-Based Access Control (RBAC): Distinct user roles (Job Seeker, Recruiter, Admin) with granular permissions. Middleware validates authorization for each request.

📊 User Story Mapping

We used story mapping to visualize the user journey and prioritize features. High-level activities (Find Job, Apply, Communicate) were broken into user stories, then tasks. This helped us identify MVP scope and plan releases.

👥 Team Roles & Organization

Following Agile methodology, our team organized into specialized roles to maximize efficiency and accountability throughout the development process:

Scrum Master: Facilitated sprint ceremonies, removed blockers, and ensured the team adhered to Agile principles. Maintained sprint backlogs and tracked velocity across iterations.

Product Manager: Defined product vision, prioritized features based on user value, and maintained the product backlog. Acted as the voice of the customer and stakeholder liaison.

Frontend Developers: Built responsive user interfaces using modern frameworks, implemented client-side logic, and ensured seamless user experiences across devices.

Backend Developers: Designed and implemented RESTful APIs, database schemas, authentication systems, and server-side business logic to power the platform.
